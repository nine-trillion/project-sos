# 기능 명세서 - 장바구니 / 주문

## 1. 장바구니 등록

### 1.1 장바구니 항목 추가

- **설명**: 사용자가 자신의 장바구니에 메뉴를 추가할 수 있다.
- **요청 데이터**:
    - 메뉴 ID
    - 수량
- **처리 로직**:
    1. 로그인 여부 확인
    2. 사용자의 장바구니 조회 (없으면 생성)
    3. 메뉴 조회 후 장바구니 항목 생성 및 저장
- **예외 처리**:
    - 로그인하지 않은 경우: `CartException(CartError.NOT_LOGGED_IN)`

## 2. 장바구니 조회

### 2.1 장바구니 항목 조회

- **설명**: 사용자가 자신의 장바구니에 담긴 항목들을 조회할 수 있다.
- **요청 데이터**:
    - 없음
- **처리 로직**:
    1. 로그인 여부 확인
    2. 장바구니 항목 리스트 조회 및 응답
- **예외 처리**:
    - 로그인하지 않은 경우: `CartException(CartError.NOT_LOGGED_IN)`

## 3. 장바구니 수정

### 3.1 장바구니 항목 수정

- **설명**: 사용자가 장바구니 항목의 수량을 수정할 수 있다.
- **요청 데이터**:
    - 장바구니 항목 ID
    - 수정할 수량
- **처리 로직**:
    1. 로그인 여부 확인
    2. 장바구니 항목 조회
    3. 소유자 검증
    4. 수량 수정
- **예외 처리**:
    - 로그인하지 않은 경우: `CartException(CartError.NOT_LOGGED_IN)`
    - 다른 사용자의 장바구니 항목 수정 시도: `CartException(CartItemError.NO_PERMISSION)`

## 4. 장바구니 삭제

### 4.1 장바구니 항목 삭제

- **설명**: 사용자가 장바구니에 담긴 항목을 삭제할 수 있다.
- **요청 데이터**:
    - 장바구니 항목 ID
- **처리 로직**:
    1. 로그인 여부 확인
    2. 장바구니 항목 조회
    3. 소유자 검증
    4. 항목 삭제
- **예외 처리**:
    - 로그인하지 않은 경우: `CartException(CartError.NOT_LOGGED_IN)`
    - 다른 사용자의 장바구니 항목 삭제 시도: `CartException(CartItemError.NO_PERMISSION)`

## 5. 주문 등록

### 5.1 주문 생성

- **설명**: 사용자가 장바구니에 담긴 메뉴들로 주문을 생성할 수 있다.
- **요청 데이터**:
    - 유저 ID
    - 가게 ID
    - 주문 항목 (메뉴 ID, 수량)
    - 총 가격
    - 요청 시간
- **처리 로직**:
    1. 로그인 여부 확인
    2. 유저, 가게 조회
    3. 주문 생성 및 저장
    4. 주문 항목 저장
- **예외 처리**:
    - 로그인하지 않은 경우: `OrderException(OrderError.NOT_LOGGED_IN)`

## 6. 주문 조회

### 6.1 주문 단건 조회

- **설명**: 사용자가 본인의 주문 상세 정보를 조회할 수 있다.
- **요청 데이터**:
    - 주문 ID
- **처리 로직**:
    1. 로그인 여부 확인
    2. 주문 조회
    3. 소유자 검증 후 주문 정보 응답
- **예외 처리**:
    - 로그인하지 않은 경우: `OrderException(OrderError.NOT_LOGGED_IN)`
    - 다른 사용자의 주문 조회 시도: `OrderException(OrderError.NO_PERMISSION)`

### 6.2 내 주문 목록 조회

- **설명**: 사용자가 자신의 모든 주문 목록을 조회할 수 있다.
- **요청 데이터**:
    - 없음
- **처리 로직**:
    1. 로그인 여부 확인
    2. 로그인한 사용자의 주문 목록 조회
- **예외 처리**:
    - 로그인하지 않은 경우: `OrderException(OrderError.NOT_LOGGED_IN)`

### 6.3 관리자 - 특정 유저 주문 목록 조회

- **설명**: 관리자가 특정 사용자의 주문 목록을 조회할 수 있다.
- **요청 데이터**:
    - 조회할 유저 ID
- **처리 로직**:
    1. 로그인 여부 확인
    2. 로그인한 사용자가 관리자 권한인지 검증
    3. 특정 사용자의 주문 목록 조회
- **예외 처리**:
    - 로그인하지 않은 경우: `OrderException(OrderError.NOT_LOGGED_IN)`
    - 관리자가 아닌 사용자가 접근 시도: `OrderException(OrderError.NO_PERMISSION)`

## 7. 주문 취소

### 7.1 주문 취소

- **설명**: 사용자가 본인의 주문을 취소할 수 있다.
- **요청 데이터**:
    - 주문 ID
- **처리 로직**:
    1. 로그인 여부 확인
    2. 주문 조회
    3. 소유자 검증 후 주문 상태 'CANCELLED'로 변경
- **예외 처리**:
    - 로그인하지 않은 경우: `OrderException(OrderError.NOT_LOGGED_IN)`
    - 다른 사용자의 주문 취소 시도: `OrderException(OrderError.NO_PERMISSION)`

## 8. 주문 상태 변경

### 8.1 주문 상태 변경

- **설명**: 사용자(OWNER)가 본인 가게의 주문 상태를 변경할 수 있다.
- **요청 데이터**:
  - 주문 ID
  - 가게 ID
  - 주문 상태 (`COOKING, DELIVERING, COMPLETED, CANCELLED`)
- **처리 로직**:
  1. 로그인 여부 확인
  2. 로그인한 사용자가 해당 가게의 사장인지 확인
  3. 주문 조회
  4. 주문 상태 변경
- **예외 처리**:
  - 로그인하지 않은 경우: `OrderException(OrderError.NOT_LOGGED_IN)`
  - 다른 사용자 가게의 주문 상태 변경 시도: `StoreException(StoreError.UNAUTHORIZED_STORE_OWNER)`
