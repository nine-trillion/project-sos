# 기능 명세서 - 메뉴

## 1. 메뉴 등록

### 1.1 메뉴 생성

- **설명**: 사장(OWNER) 권한을 가진 사용자가 가게에 메뉴를 추가할 수 있다.
- **요청 데이터**:
    - 가게 ID
    - 메뉴 이름
    - 가격
    - 카테고리
- **처리 로직**:
    1. 요청에 포함된 권한(role) 값이 `OWNER`인지 확인
    2. 가게 ID, 메뉴명, 가격, 카테고리 유효성 검증
    3. 메뉴 엔티티 생성 및 저장
    4. 저장된 메뉴 정보를 응답
- **예외 처리**:
    - 권한이 없는 경우: `MenuException(MenuError.NO_PERMISSION)`
    - 입력값이 유효하지 않은 경우: `MethodArgumentNotValidException`

## 2. 메뉴 조회

### 2.1 가게별 카테고리별 메뉴 목록 조회

- **설명**: 가게 ID와 카테고리명을 기준으로 해당 가게의 메뉴들을 조회할 수 있다.
- **요청 데이터**:
    - 가게 ID
    - 카테고리명
- **처리 로직**:
    1. 가게 ID와 카테고리로 메뉴 목록 조회 (삭제되지 않은 메뉴만)
    2. 조회된 메뉴 리스트를 응답
- **예외 처리**:
    - 별도 예외 없음 (조회 결과가 없으면 빈 리스트 반환)

## 3. 메뉴 수정

### 3.1 메뉴 정보 수정

- **설명**: 사장(OWNER) 권한을 가진 사용자가 메뉴 정보를 수정할 수 있다.
- **요청 데이터**:
    - 메뉴 이름
    - 가격
    - 카테고리
- **처리 로직**:
    1. 요청에 포함된 권한(role) 값이 `OWNER`인지 확인
    2. 메뉴 ID로 기존 메뉴 조회
    3. 메뉴 이름, 가격, 카테고리를 업데이트
    4. 수정된 메뉴 정보를 응답
- **예외 처리**:
    - 권한이 없는 경우: `MenuException(MenuError.NO_PERMISSION)`
    - 메뉴를 찾을 수 없는 경우: `MenuException(MenuError.MENU_NOT_FOUND)`
    - 입력값이 유효하지 않은 경우: `MethodArgumentNotValidException`

## 4. 메뉴 삭제

### 4.1 메뉴 삭제 (Soft Delete)

- **설명**: 사장(OWNER) 권한을 가진 사용자가 메뉴를 삭제할 수 있다. (DB에서는 삭제되지 않고 isDeleted 플래그만 변경)
- **요청 데이터**:
    - 메뉴 ID
- **처리 로직**:
    1. 권한(role)을 `OWNER`로 가정하여 검증
    2. 메뉴 ID로 메뉴 조회
    3. 메뉴 엔티티의 `isDeleted` 값을 `true`로 변경
- **예외 처리**:
    - 권한이 없는 경우: `MenuException(MenuError.NO_PERMISSION)`
    - 메뉴를 찾을 수 없는 경우: `MenuException(MenuError.MENU_NOT_FOUND)`
